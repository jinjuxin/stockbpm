# StockBPM

这是一个按中文目录整理的 AI 协作工作区。

## 顶层结构

```text
智能体/
技能/
文档/
应用/
AGENTS.md
README.md
```

## 推荐组织方式

- `智能体/`：只保留一个`主控智能体`
- `技能/`：按阶段拆分的执行模板
- `文档/流程/`：流程、门禁、协作规则
- `文档/规范/`：系统级约束模板
- `文档/模板/`：阶段产物模板
- `文档/需求/`：每个需求的完整链路产物
- `应用/`：可运行页面或系统

## 为什么这样拆

当前更合理的模式不是“很多智能体互相调度”，而是：

- 一个主控智能体：判断阶段、决定回退还是推进
- 多个技能：负责每个阶段怎么做

这样能减少重复定义，也更适合你后面逐个调试。
