# 开发标准模板

这是一套可复制到任意项目的轻量开发标准模板，目标是先固化流程与产物，再逐步演进为 skill 和 agent。

适用流程：

1. 需求
2. 原型
3. 原型确认
4. 技术方案
5. 方案确认
6. 开发
7. 测试
8. 验收
9. 文档

目录说明：

- `docs/process/`：流程与门禁说明
- `docs/templates/`：各阶段输出模板
- `docs/project/`：项目约束与边界模板
- `skills/`：后续演进为 skill 的骨架

建议使用方式：

1. 先复制整个目录到目标项目
2. 根据项目实际情况补充模块边界、技术约束、命名规范
3. 先用模板跑通 3 到 5 个真实需求
4. 模板稳定后，再抽成 skill
5. 最后再让 agent 按流程调用
