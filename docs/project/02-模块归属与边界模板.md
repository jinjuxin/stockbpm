## 3. 归属判断规则

当新增需求时，按以下顺序判断：

1. 是否为多个业务域复用的公共能力
2. 是否为技术支撑能力
3. 是否为独立业务域能力
4. 是否只是某业务模块内部实现细节

## 5. 新建模块判断

以下情形可考虑新建模块：

- 业务语义独立
- 生命周期独立
- 有独立数据模型
- 后续扩展性明显

以下情形不建议新建模块：

- 只是现有模块的小扩展
- 只有一两个简单接口
- 还没有形成稳定领域边界

## 6. 常见反例

- 为了开发方便，把技术能力放进业务模块
- 为了省事，把多个不相干能力都堆到一个模块
- 为了临时联调，直接跨模块调用内部实现
