# 需求澄清 Skill 骨架

## 1. 适用场景

- 用户给出一个功能需求
- 需求描述不完整，需要补齐背景、范围和验收标准

## 2. 输入

- 原始需求描述
- 相关背景材料
- 现有流程或历史问题

## 3. 输出

- 一份完整的需求任务卡

## 4. 必查项

- 为什么做
- 给谁用
- 解决什么问题
- 不做什么
- 验收标准是什么

## 5. 禁止事项

- 只记录功能名称，不记录背景
- 只有模糊目标，没有明确验收标准
- 混入技术方案内容

## 6. 执行步骤

1. 提取需求目标
2. 提取目标用户与场景
3. 识别范围与非目标
4. 提取业务规则
5. 补齐验收标准
6. 输出任务卡
