# Skill 骨架说明

这里不是某个平台专属 skill 格式，而是通用骨架。

目标是把稳定下来的文档和模板，进一步抽象成可执行的阶段说明，后续可迁移到不同工具：

- Codex
- Claude Code
- Antigravity
- OpenClaw
- 其他 agent 编排系统

建议演进顺序：

1. 先用 `docs/` 里的流程和模板跑真实项目
2. 哪个步骤重复 5 次以上，就抽成 skill
3. skill 先写成普通 markdown
4. 等动作稳定后，再转成平台专属格式

当前建议优先沉淀的 skill：

- 需求澄清
- 技术方案
- 测试验收

每个 skill 骨架都建议包含：

- 适用场景
- 输入
- 输出
- 必查项
- 禁止事项
- 执行步骤
