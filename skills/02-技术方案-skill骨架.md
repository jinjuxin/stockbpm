# 技术方案 Skill 骨架

## 1. 适用场景

- 原型已确认，需要进入技术设计
- 需要判断模块归属、接口边界、配置与基础设施影响

## 2. 输入

- 需求任务卡
- 已确认原型
- 项目约束文档

## 3. 输出

- 一份技术方案说明

## 4. 必查项

- 模块归属
- 数据结构
- 接口边界
- 权限影响
- 配置影响
- 缓存、MQ、任务是否涉及
- 风险与回滚方案

## 5. 禁止事项

- 原型未确认就直接设计接口
- 不判断模块归属就开始编码
- 忽略权限、配置、网关、消息等外围影响

## 6. 执行步骤

1. 判断模块归属
2. 列出改动范围
3. 设计数据和接口
4. 判断基础设施影响
5. 列出风险与回滚方式
6. 输出技术方案
