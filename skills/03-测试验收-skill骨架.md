# 测试验收 Skill 骨架

## 1. 适用场景

- 开发完成后，需要进行代码级测试和运行级验收

## 2. 输入

- 代码改动
- 需求任务卡
- 已确认原型
- 技术方案

## 3. 输出

- 测试清单
- 验收清单
- 问题记录

## 4. 必查项

- 编译是否通过
- 关键测试是否通过
- 原型主流程是否跑通
- 异常场景是否验证
- 验收标准是否满足

## 5. 禁止事项

- 编译不过直接提验收
- 只测 happy path
- 只凭主观感觉判断完成

## 6. 执行步骤

1. 跑代码级测试
2. 记录结果与问题
3. 跑关键业务流程
4. 对照验收标准逐项核查
5. 保留验收证据
6. 输出测试和验收结论
