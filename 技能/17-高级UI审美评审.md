# 高级UI审美评审

## 作用

- 在原型生成后，从高级官网和高审美 UI 的角度进行批判式评审
- 识别页面是否存在廉价感、模板感、用力过猛或不够高级的问题

## 适用场景

- 官网首页
- 品牌站
- 高保真营销页
- 用户明确提出“高级感、大气、像大厂、像设计总监压过”的要求

## 输入

- 页面原型
- 品牌视觉策略
- 页面文案与模块结构

## 输出

- 审美问题清单
- 高级感风险说明
- 优先级明确的优化建议

## 评审维度

### 1. 颜色

- 是否看起来廉价
- 是否太艳、太土、太普通
- 是否缺少真正有记忆点的色彩策略

### 2. 版式

- 是否像通用模板
- 是否太碎
- 是否没有主次与层级张力

### 3. 组件

- 卡片、按钮、标签是否过于普通
- 阴影、圆角、边框是否缺少克制感

### 4. 文案与视觉关系

- 文案是否太解释型
- 页面是否像“说明书”而不是“品牌站”

### 5. 统一性

- 首页与内页是否同属一套成熟设计语言

## 禁止事项

- 不用“还可以”“挺好”这类模糊判断
- 不把明显普通的设计描述成高级
- 不只给抽象意见，必须指出是哪一类问题
