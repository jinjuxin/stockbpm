# 需求任务卡

## 1. 基本信息

- 需求编号：需求-003
- 需求名称：脱骨侠集团官网
- 提出人：当前项目负责人
- 优先级：高
- 提出日期：2026-04-20
- 需求类型：新系统需求
- 是否基于现有系统：否
- 是否要求风格一致：不适用
- 是否涉及技术框架搭建：待确认

## 2. 背景

- 当前问题：脱骨侠集团缺少一套正式的集团官网需求定义，无法直接进入结构设计、原型和开发阶段。
- 为什么现在做：需要先把集团官网第一版的目标、页面范围和内容边界梳理清楚，确保后续设计和实现围绕品牌展示展开。
- 如果不做会怎样：后续容易直接进入页面设计，导致栏目结构、品牌表达和内容重点不一致，返工概率高。

## 3. 目标

- 目标 1：搭建一套用于品牌展示的集团官网信息架构。
- 目标 2：明确第一版官网的页面范围、栏目边界和内容重点。
- 目标 3：为后续结构图、原型和官网实现提供稳定输入。

## 4. 范围

### 包含范围

- 1. 首页
- 2. 集团介绍
- 3. 业务板块
- 4. 新闻中心
- 5. 社会责任
- 6. 招聘
- 7. 联系我们

### 不包含范围

- 1. 电商交易能力
- 2. 经销商后台或管理后台
- 3. 会员中心或用户登录体系
- 4. 复杂互动活动系统

## 5. 用户与场景

- 目标用户：消费者、合作伙伴、求职者、媒体与社会公众
- 使用角色：官网访客
- 典型场景：
  - 了解脱骨侠集团品牌与业务
  - 查看企业新闻与社会责任内容
  - 了解招聘信息并建立联系
- 使用频率与时机：品牌传播、商务合作了解、求职了解、媒体查阅时访问
- 使用端：Web
- 语言版本：中文、越南语

## 6. 业务规则

- 规则 1：官网第一版以品牌展示为核心，不承载交易闭环。
- 规则 2：首页与一级栏目必须围绕集团品牌形象、业务实力和企业内容展开。
- 规则 3：页面结构需支持后续继续扩展新闻、招聘和社会责任内容。

## 7. 影响对象

- 页面或端：集团官网 Web 端
- 模块：品牌展示、栏目导航、内容展示、联系入口
- 数据对象：集团介绍内容、业务板块内容、新闻内容、社会责任内容、招聘内容、联系信息
- 接口：待确认
- 权限：默认前台公开浏览

## 8. 验收标准

- 验收项 1：能明确官网第一版的信息架构和页面范围。
- 验收项 2：能围绕“品牌展示”形成清晰的结构图和页面原型输入。
- 验收项 3：一级栏目和首页内容重点不跑偏，能支撑后续设计和实现。

## 9. 风险与待确认项

### 已知风险

- 风险 1：官网需要兼顾“食品集团企业感”和“消费品牌年轻化表达”，如果平衡不好，原型阶段容易出现风格偏差。
- 风险 2：如果没有明确每个栏目展示重点，后续页面信息密度和层级可能失衡。

### 待确认项

- 待确认 1：招聘页是只展示招聘信息入口，还是要做完整岗位列表与投递入口？
- 待确认 2：新闻中心是否需要区分“集团新闻 / 媒体报道 / 公告”？
- 待确认 3：社会责任页面更偏公益展示、食品安全，还是可持续发展内容？
- 待确认 4：中文与越南语是否要求完全一一对应的双语切换？

## 10. 备注

- 参考材料：当前对话输入

- 其他说明：

  - 官网调性按“食品集团企业感 + 消费品牌年轻化”结合处理
  - 首页内容重点优先突出产品/业务实力
