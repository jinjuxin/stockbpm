# 原型说明

## 1. 原型目标

本轮原型只解决“页面怎么承载已明确的业务内容”，不再重复业务需求分析。

当前原型聚焦第一版必做范围：

- 工单列表
- 新建工单
- 工单详情
- 分派处理
- 搜索筛选
- 状态流转入口
- 处理记录展示

## 2. 页面清单

### 2.1 列表页

用途：
- 查看工单列表
- 搜索与筛选
- 进入详情页
- 管理员进入分派页
- 新建工单入口

展示字段：
- 工单编号
- 标题
- 优先级
- 提交人
- 处理人
- 状态
- 创建时间

筛选项：
- 标题关键词
- 状态
- 优先级
- 处理人

### 2.2 新建页

用途：
- 提交人创建工单

录入字段：
- 标题
- 内容
- 优先级

只读字段：
- 提交人
- 初始状态
- 创建时间

### 2.3 详情页

用途：
- 查看工单完整信息
- 展示处理记录
- 根据角色显示不同操作

展示字段：
- 工单编号
- 标题
- 内容
- 提交人
- 处理人
- 优先级
- 创建时间
- 当前状态

### 2.4 分派页

用途：
- 管理员为工单指定处理人

字段：
- 当前工单标题
- 当前状态
- 当前处理人
- 新处理人
- 分派备注

## 3. 角色界面差异

### 提交人

- 只看自己提交的工单
- 可新建工单
- 可查看详情
- 不可分派
- 不可推进状态

### 处理人

- 只看分派给自己的工单
- 可查看详情
- 可推进状态
- 不可分派

### 管理员

- 可查看全部工单
- 可分派处理人
- 可推进状态

## 4. 状态流转

固定顺序：

`待处理 -> 处理中 -> 已完成 -> 已关闭`

原型要求：
- 不展示跳级入口
- 已关闭后不展示继续推进按钮

## 5. 页面跳转

- 列表页 -> 新建页
- 列表页 -> 详情页
- 列表页 -> 分派页
- 新建页 -> 详情页
- 新建页 -> 列表页
- 详情页 -> 列表页
- 详情页 -> 分派页
- 分派页 -> 详情页

## 6. 当前结论
