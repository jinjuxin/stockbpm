# 原型确认记录

## 1. 基本信息
- 需求名称：工单系统 MVP
- 原型版本：v1.0
- 评审日期：2026-04-18
- 评审参与人：需求提出人、开发实现方

## 2. 原型范围
- 涉及页面：工单列表页、新建工单页、工单详情页、管理员分派区域
- 涉及角色：提交人、处理人、管理员
- 涉及流程：创建工单 -> 管理员分派 -> 处理人处理 -> 完成 -> 关闭

## 3. 已确认结论
- 提交人默认只看自己提交的工单。
- 处理人默认只看分派给自己的工单。
- 管理员可查看全部工单。
- 管理员可以分派处理人，也可以直接推进状态。
- 工单编号自动生成，格式示例：`TK20260418-001`。
- 优先级固定为：低 / 中 / 高 / 紧急。
- 状态流转固定为：待处理 -> 处理中 -> 已完成 -> 已关闭。
- 不允许跳级流转，已关闭后默认不可编辑。
