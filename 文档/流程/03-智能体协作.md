# 智能体协作

## 当前采用的模型

- 一个主控智能体
- 多个阶段技能

## 当前可调度技能

- 主控调度
- 需求澄清
- 业务需求拆解
- 存量系统对齐
- 原型生成
- 原型评审
- 技术方案
- 开发实现
- 测试验收
- 文档回写
- 需求完整性检查
- 需求冲突检查
- 非功能需求分析
- 需求评审结论
- 需求追问策略

## 标准路径

### 1. 系统级需求

`需求追问策略(按需) -> 需求澄清 -> 业务需求拆解 -> 需求确认图 -> 需求完整性检查 -> 需求冲突检查 -> 非功能需求分析 -> 需求评审结论 -> 原型生成 -> 原型评审 -> 技术方案 -> 开发实现`

### 2. 小需求

`需求追问策略(按需) -> 需求澄清 -> 需求确认图(按需) -> 需求完整性检查(按需) -> 需求评审结论(按需) -> 原型生成 -> 原型评审 -> 技术方案 -> 开发实现`

### 3. 存量系统新增

`需求追问策略(按需) -> 需求澄清 -> 存量系统对齐 -> 需求确认图 -> 需求完整性检查 -> 需求冲突检查 -> 非功能需求分析(按需) -> 需求评审结论 -> 原型生成 -> 原型评审 -> 技术方案 -> 开发实现`

## 关键原则

- 需求阶段负责业务分析
- 需求确认图负责结构、流程、状态的图形化确认
- 检查类技能负责查漏、识别冲突、补足非功能约束
- 评审结论负责裁决是否放行到下一阶段
- 原型阶段负责页面设计
- 不让原型阶段替代需求分析
